Ruby vs JavaScript

Ruby Hashes and JavaScript Objects

August 1, 2015

As you can see from the image above, one of the main differences between Ruby and JavaScript that I found very annoying was adding semi-colons to the end of almost every single line of code. The only time you don't have to add it is for `for` loops, `if` statements, `while` loops, and a couple more. But you do have to end with a semi-colon for the block of code inside those loops or statements. If you forget the semi-colon, there will be an error when you're running your code. It's an easy fix, but annoying.

Another thing that was different (and less annoying) was that you needed to add these: `{}` for your `if` statements or loops or what have you if it's longer than one line. In Ruby, if you indent it, the code will understand that the three lines following those things are part of it. But for JavaScript, the block of code needs to be between those squiggly brackets.

Let's move away from those small differences. Just had to vent that out. One thing we should look at is the difference between Ruby hashes and JavaScript objects.
