Arrays and Hashes - When Do I Use These

June 27, 2015

It's the third week at DBC and this week was a lot easier. Started an introduction to Ruby, which was an easy concept to understand due to the fact that I have taken an introductory C++ class in college back in 2006. It's a long time ago, but everything I learned came back to me while learning about Ruby. So many similarities and so many differences as well.

One of the concepts we learned was about arrays. It's not even a word you hear every day like padding, border, or the other terms used in CSS. So I'm here to explain what it is.

To begin, when writing in Ruby or any other language, information is usually stored in a variable and these variables are named however you want them. Usually it's something that tells what information is stored in the variable. For example you might call a variable "name" because it stores names. The most important thing is that each variable can only store one piece of information. So the names Alfred and Ben cannot be stored in the variable "name". You would have to create another variable "name2" or however you want to name them.

But let's say you have a lists of names. You wouldn't want to create variables "name1", "name2", "name3" and so forth. It's just so much work to type. In addition, what if you want to add more names to that list later? The program has no way of knowing how many more variables you'll need. This is where arrays come in. An array is just a variable that stores different arguments in its many slots. Just imagine a row with infinitely many columns. Each column is stored with a different argument. Now you're wondering how we get each element if they are stored in a different space. But first, let's look at how we assign the elements to each slot.

Arrays seems easy enough right? But what if you have the office's lunch order? Alfred wants a roast beef sandwich, Ben wants a turkey sandwich, Connor wants pastrami sandwich, and so forth. If you use an array, you would need one array for the names and another array for the order. But there is a better way with hashes. Hashes are arrays, but with a minor difference. With arrays, numbers are assigned for each column as seen with [0] or [2]. But with hashes, you can assign whatever you want in between each square bracket [].

With hashes, it's a lot simpler to get the information you need compared to arrays. With an array, you can think to yourself "What did Ben order?" and you might not be sure if you put his order in the 0th slot or the 3rd slot of the array so you don't know where to get the information. But with hashes, you can get it from the "Ben" slot with no problem.
